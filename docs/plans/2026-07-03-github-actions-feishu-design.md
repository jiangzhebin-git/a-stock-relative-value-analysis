# GitHub Actions 与飞书通知设计

工作流部署在私有 GitHub 仓库，每个交易日北京时间 15:35 定时运行，也允许手动触发。任务安装 Python 依赖、运行测试、更新四股数据与模型、读取去重后的 `new_alerts.csv`，仅在产生新买卖信号时调用飞书机器人 Webhook。分析程序失败时执行独立故障通知，避免“没有消息”被误解为“没有信号”。

历史数据、模型结果、报告和 `alert_state.json` 提交回仓库，作为下一次运行的缓存与去重状态。飞书 Webhook 仅存储在 GitHub Actions Secret `FEISHU_WEBHOOK_URL` 中，不进入代码、日志或产物。仓库工作流仅申请 `contents: write` 权限。

由于 GitHub 托管执行器访问国内行情接口可能波动，分析层保留东方财富、新浪、腾讯三级日线降级和本地缓存。分钟线失败不阻断日线任务；通知中明确数据日期。工作流提交结果前检查实际文件变化，避免空提交和无意义触发。
