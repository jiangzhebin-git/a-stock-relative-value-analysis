# 东方财富相对价值分析工具设计

## 定位与边界

本工具判断东方财富（300059）相对于大盘、证券板块和市场成交活跃度的相对贵贱，不预测确定性涨跌，不自动交易。第一版使用日线数据，所有交易信号至少延迟到下一交易日执行，避免未来数据泄漏。

## 数据与特征

数据层优先使用 AkShare：A 股复权日线、主要指数、证券板块代表指数，以及沪深市场成交额。原始数据分别缓存为 CSV；获取失败时保留明确日志并复用本地缓存。预处理以交易日期内连接，收益率和变化率统一在各自序列内计算，极端变化率按训练集分位数缩尾。

模型目标是东方财富日收益率。模型 A-D 逐步加入主要指数、证券板块、两市成交额和个股量能。采用 Ridge 回归降低指数之间高度共线性的影响，使用扩展窗口时间序列验证产生样本外预测。最终模型综合样本外 R²、MAE、RMSE、方向准确率和回测表现选择，不以单一指标拍板。

## 信号、回测与报告

理论价格等于前收盘价乘以样本外理论收益率。偏离指数基于当日实际收盘价与理论价格计算。买入、卖出信号结合偏离阈值、证券板块趋势、两市成交额和个股量能过滤。回测在信号后的下一交易日开盘执行，包含止损、止盈、模型回归和平仓过滤，暂不模拟融资、做空和自动下单。

挂单区间由 ATR、20 日收益波动率和历史偏离分位数共同确定，避免固定百分比。输出包含清洗数据、模型结果、相关与滞后分析、交易明细、每日分析、PNG 图表及 Markdown 报告。任何结果都必须带固定风险提示。

## 异常处理与验证

网络数据缺失时优先读取缓存；关键序列仍缺失则快速失败并说明具体标的。测试使用合成数据验证无未来函数的特征、理论价格、偏离分级、挂单区间和回测记账。运行入口支持日期、数据目录和输出目录参数，可重复运行。
