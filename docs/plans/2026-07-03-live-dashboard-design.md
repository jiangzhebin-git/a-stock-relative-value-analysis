# 四股动态偏离看板设计

## 单一算法来源

每日收盘流水线训练四组候选模型并发布 `model_registry.json`。注册表记录每只股票的正式模型、有效系数、前收盘、波动区间、训练数据截止日、样本外指标和版本号。飞书通知、日报和盘中网页都必须消费这份注册表，不允许各自复制阈值或参数。

每日可以生成候选参数，但正式参数采用稳定性门槛：样本外指标完整、交易样本充足度有标注、模型相对上一版没有异常退化。每周生成参数变化和年度表现摘要。第一版保留完整历史扩展窗口训练；后续注册表可以增加“候选/正式”双版本晋级。

## 盘中刷新

盘中任务只刷新市场输入，不重训。实时理论收益由正式模型系数乘以当前指数、证券板块、成交活跃度和个股量能特征得到。累计成交额与成交量按交易时间进度折算成预计全天值，再和前一完整交易日比较。缺少某项实时数据时使用中性值并降低置信等级；所有实时接口失败则回退至最近日线结果。

GitHub Actions在交易时间约每10分钟运行轻量任务并发布 `live_snapshot.json`。GitHub定时任务可能延迟，所以网页必须展示采集时间、数据年龄、动态/静态模式和置信等级。

## 网页交互

页面采用深色金融终端风格，登录前显示轻量PIN遮罩。PIN仅作访问阻挡，不属于强安全认证。登录后直接显示四张股票卡片：当前价格、动态理论价、偏离率、贵贱判断、买卖三区、风险和数据状态。顶部只有刷新按钮和最后更新时间；按钮强制绕过浏览器缓存读取最新快照，不要求用户输入价格。

移动端单列、桌面端两列。加载、接口失败、快照过期和PIN错误都有明确状态。页面不保存持仓、资金、Webhook或其他敏感信息。
